⚡ SNI Spoofing Cheker
یک ابزار قدرتمند، سریع و گرافیکی (GUI) که با استفاده از زبان پایتون توسعه یافته است. این برنامه به شما اجازه می‌دهد تا ساب‌نت‌های آی‌پی (IP Subnets) را با استفاده از تکنیک SNI Spoofing اسکن کرده و آی‌پی‌هایی که اتصال TLS/SSL سالمی با SNI هدف دارند را به سرعت پیدا کنید.

این ابزار با بهره‌گیری از asyncio و پردازش ناهمگام (Asynchronous)، سرعت فوق‌العاده‌ای در اسکن شبکه دارد و رابط کاربری مدرن آن، کار را برای هر کاربری آسان می‌کند.

✨ ویژگی‌های کلیدی (Features)
🎨 رابط کاربری مدرن و تاریک: طراحی جذاب (Cyberpunk/Dark Theme) با استفاده از Tkinter که تجربه کاربری فوق‌العاده‌ای ارائه می‌دهد.

🚀 سرعت اسکن بسیار بالا: استفاده از کتابخانه asyncio برای بررسی همزمان صدها آی‌پی (Concurrency) بدون افت عملکرد.

📊 نمایش زنده نتایج (Real-Time): مشاهده آی‌پی‌های پیدا شده به همراه SNI هدف و زمان پینگ (Ping) به صورت لحظه‌ای در جدول برنامه.

⚙️ پیکربندی کاملاً انعطاف‌پذیر: امکان تنظیم میزان Timeout و تعداد کانکشن‌های همزمان (Concurrency) برای بهینه‌سازی اسکن بر اساس قدرت اینترنت شما.

📁 پشتیبانی از ورودی‌های چندگانه: قابلیت دریافت چندین ساب‌نت (با فرمت CIDR) و چندین دامنه SNI به صورت همزمان.

💾 خروجی خودکار: ذخیره‌سازی خودکار و مرتب‌شده‌ی تمام آی‌پی‌های سالم در یک فایل متنی (working_tls_ips.txt) در پایان هر اسکن.

🛡️ پایداری بالا: مدیریت هوشمند خطاها و جلوگیری از فریز شدن (هنگ کردن) برنامه هنگام اسکن‌های سنگین با استفاده از Threading.

پیش‌نیازها (Prerequisites)
برای اجرای این برنامه، نیازی به نصب پکیج‌های خارجی سنگین ندارید، چرا که اکثر ماژول‌های استفاده شده در پایتون به صورت پیش‌فرض وجود دارند.
تنها چیزی که نیاز دارید:

Python 3.8 یا نسخه‌های جدیدتر

سیستم عامل ویندوز، لینوکس یا مک (در لینوکس ممکن است نیاز به نصب بسته python3-tk داشته باشید).

🛠️ راهنمای استفاده از برنامه
پس از باز شدن پنجره نرم‌افزار، با بخش‌های زیر روبه‌رو می‌شوید:

1. تنظیمات (Configuration)
Timeout (s): حداکثر زمانی که برنامه منتظر پاسخ از سمت یک آی‌پی می‌ماند (پیش‌فرض: 1.5 ثانیه). اگر اینترنت کندی دارید، این عدد را کمی بالاتر ببرید.

Concurrency: تعداد پردازش‌های همزمان. هرچه این عدد بیشتر باشد سرعت اسکن بالاتر می‌رود، اما به منابع سیستم و اینترنت بیشتری نیاز دارد (پیش‌فرض: 150).

2. ورودی‌ها (Inputs)
IP Subnets Target: محدوده‌های آی‌پی که می‌خواهید اسکن شوند. می‌توانید از فرمت CIDR استفاده کنید (مانند 104.16.0.0/24) و در هر خط یک ساب‌نت قرار دهید.

SNI Hostnames: آدرس‌های SNI که می‌خواهید تست شوند (مانند www.cloudflare.com). برنامه تک تک آی‌پی‌های تولید شده را با این دامنه‌ها چک می‌کند.

3. اجرا و نتایج
روی دکمه START SCAN کلیک کنید. برنامه شروع به اسکن کرده و نتایج را همراه با پینگ در ترمینال پایین صفحه لیست می‌کند. هر زمان که بخواهید می‌توانید با دکمه STOP SCAN فرآیند را متوقف کنید. پس از پایان اسکن، آی‌پی‌های سالم در فایلی به نام working_tls_ips.txt در کنار فایل اصلی برنامه ذخیره می‌شوند.
